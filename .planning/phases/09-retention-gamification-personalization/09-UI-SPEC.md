## Interaction Design

### SRS Review Room (`/review`)
- **Layout:** Centered single card, max-w-md, vertically centered on page.
- **Card content:** Word (large display style) + translation (body style below) + lyric snippet (muted italic, bottom of card).
- **No flip interaction** — word and translation are visible simultaneously.
- **3 rating buttons:** Horizontal row below card content. Colorless (uses standard secondary button variant). Mapped to SM-2 scores: Good=5, Hard=3, Again=1.
- **Progress:** "3/12" header badge showing completed/total for this session.
- **End early:** "End review" link below rating row. Saves partial progress, redirects to dashboard.
- **Completion:** When all words reviewed, transitions to SRSReviewEmpty state with celebration.

### Playlist Deletion
- **No confirmation dialog.** Inline toast approach.
- **Toast:** Bottom-center, 5s auto-dismiss. "Playlist deleted. Undo?" with clickable "Undo" label.
- On "Undo": calls restore API, toast replaced with "Playlist restored" (2s auto-dismiss).
- **Implementation:** Toast component (inline, no library). Uses `useState` + `useEffect` timer.

### Badge Unlock Flow
- **Detection:** Server returns `badges_unlocked[]` in relevant API responses (quiz complete, playlist create, etc.).
- **Notification:** Toast banner at top-right, 4s auto-dismiss. Shows badge icon (colored) + "{Badge Name} unlocked!".
- **Dashboard card:** Auto-refreshes on next visit. Newly unlocked badges transition from grayscale to color.

### Onboarding Flow
- **Route:** `/onboarding` — standalone page, no nav.
- **Step 1:** Two `<select>` fields: Native language + Target language (empty by default, no pre-fill).
- **Step 2:** Genre multi-select + Difficulty slider/radio (optional).
- **Skip:** "Skip → Use defaults" link at bottom (EN → ES, default genre/difficulty).
- **Navigation:** Previous/Next buttons. Step 1 must complete to proceed (unless skip).
- **Completion:** Redirect to `/dashboard`, language shown in nav badge.

### Language Badge (Nav)
- **Position:** In sticky nav, next to email display.
- **Format:** `{native} → {target}` (e.g., `EN → ES`).
- **Style:** Small pill, `text-[10px] font-bold uppercase tracking-widest`, border-zinc-800, bg-black/50.
- **Click:** Links to settings (future phase) or modal for language change.

### Dashboard Integration
- **Badges card:** New card in the 3-column grid (replaces or joins existing cards). Contains BadgeGrid + BadgesEmptyState.
- **Playlists card:** New card or sidebar section. Shows recent playlists + PlaylistEmptyState.
- **Review count badge:** In the Stats card or as a standalone compact card row above the grid. Shows "5 words to review today →" linking to `/review`.
- **Nav language badge:** Added to existing nav element set.
