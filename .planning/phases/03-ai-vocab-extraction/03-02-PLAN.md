<task type="auto">
  <name>Task 1: Vocab Repository & Logic</name>
  <files>server/routes/vocab.js, server/routes/vocab.test.js</files>
  <action>
    Create `server/routes/vocab.js` and implement logic to:
    1. Check if vocabulary already exists in `song_vocab_map` for a given `songId`.
    2. If not, fetch lyrics, call `aiService.extractVocabulary` (using user's `cefr_level`), and then `alignment.mapVocabToLyrics`.
    3. Save results to `vocab_items` and `song_vocab_map` using a transaction. 
       - `song_vocab_map` insert should include `line_index` and `char_start`.
       - Words that failed to map should still be returned (as "unmapped").
    4. Export a router with `GET /:songId` that returns:
       ```json
       {
         "mapped": [{ "vocab_id", "word", "definition", "line_index", "char_start", ... }],
         "unmapped": [{ "vocab_id", "word", "definition", ... }]
       }
       ```
    5. Write tests in `server/routes/vocab.test.js` to verify caching, extraction flow, and correct handling of mapped/unmapped items.
  </action>
  <verify>
    <automated>npm --prefix server test server/routes/vocab.test.js</automated>
  </verify>
  <done>Vocab logic handles caching and extraction correctly, distinguishing between mapped and unmapped words.</done>
</task>
